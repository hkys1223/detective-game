# 🔍 IP.107.SECURE - 디지털 방어의 논리적 시각화

> **"지능화되는 사이버 범죄, 우리는 그 이면의 논리를 이해하고 있는가?"**
> 본 프로젝트는 급증하는 사이버 테러와 개인정보 유출이라는 사회적 위협 속에서, 복합적인 보안 관제 프로세스를 **'이기종 상태 관리 아키텍처'**로 풀어낸 기술 실증형 프로젝트입니다.

---

## 🌐 Background: 프로젝트의 사회적 가치

### 1. 보안 문해력(Security Literacy)의 부재
현대 사회에서 사이버 공격은 일상화되었지만, 대중은 보안 시스템이 작동하는 내부 논리를 알기 어렵습니다. 본 프로젝트는 범인의 IP 주소를 추적하는 과정을 **게이미피케이션(Gamification)**하여, 보안 사고 대응의 절차와 논리적 사고의 중요성을 전달하고자 합니다.

### 2. 기술적 도전: '파편화된 기능의 단일 통제'
단순한 게임을 넘어, 성격이 완전히 다른 7가지의 보안 모듈(미니게임)을 하나의 중앙 엔진(`GameManager`)으로 오차 없이 통제하는 것을 목표로 삼았습니다.

---

## 🛠 System Architecture

### 1. [Polymorphism] MiniGame 인터페이스 규격화
서로 다른 로직을 가진 기능들을 동일한 방식으로 다루기 위해 다형성을 극대화했습니다.
* **추상화**: 모든 보안 모듈은 `MiniGame` 인터페이스를 상속받아 성공 보상(`getRewardClue`), 실패 페널티(`getPenalty`) 등의 공통 규격을 구현합니다.
* **확장성**: 새로운 보안 위협(미니게임)이 추가되어도 메인 엔진의 수정 없이 즉시 시스템에 편입될 수 있는 개방-폐쇄 원칙(OCP)을 지향합니다.

### 2. [Centralized Control] GameManager 엔진
게임의 모든 상태(시간, 경보, 턴, 단서)를 중앙에서 관리하는 **Single Source of Truth** 원칙을 준수합니다.
* **상태 동기화**: 미니게임의 결과가 발생하는 즉시 메인 HUD로 이벤트를 발행하여 데이터 무결성을 보장합니다.

## 🚀 트러블슈팅 (Troubleshooting)

### 1. Swing 컴포넌트의 테마 일괄 적용 문제
* **문제 상황**: Java Swing은 기본 시스템 테마를 따르기에, '보안/해킹' 컨셉의 다크 모드를 수많은 패널과 버튼에 일일이 적용하는 것은 유지보수 측면에서 매우 비효율적이었습니다.
* **해결 방안**: **재귀적 탐색 알고리즘(Recursive Component Search)**을 도입했습니다. `setDarkTheme(Component component)` 메서드를 설계하여, 최상위 프레임부터 하위 자식 노드까지 자동으로 탐색하며 일괄적으로 색상과 테두리를 적용하도록 구현했습니다.
* **결과**: 코드 가독성을 확보함과 동시에, 새로운 UI 요소가 추가되어도 일관된 사용자 경험(UX)을 즉각 제공할 수 있게 되었습니다.

### 2. 동적 시나리오 생성 및 논리적 추론 설계
* **문제 상황**: 고정된 정답은 시뮬레이션의 긴장감을 떨어뜨리고 재플레이 가치를 낮춥니다.
* **해결 방안**: `generateRandomScenario()` 메서드를 통해 매 세션마다 **랜덤 IP 주소와 연동된 논리 힌트**를 자동 생성합니다.
    * **구현 디테일**: IP의 특정 조각을 HTTP 기본 포트(80)와의 연산 값으로 변환하거나, 십의 자리와 일의 자리를 분리한 힌트를 제공하는 등 산술적 추론 과정을 강화했습니다.

### 3. 멀티 타이머 동기화 및 자원 최적화
* **문제 상황**: 메인 게임 타이머와 개별 미니게임 타이머(`TimerTask`, `javax.swing.Timer`)가 혼재된 상황에서 화면 전환 시 자원 점유 이슈가 발생할 가능성이 있었습니다.
* **해결 방안**: 각 패널이 전환되는 `showScreen()` 시점에 현재 진행 중인 타이머의 상태를 체크하고, 명시적으로 `stop()` 및 초기화 처리를 수행하는 관리 로직을 구축했습니다.
* **메모리 구조 반영**: 시스템의 안정적인 작동을 위해 데이터가 적재되는 올바른 메모리 레이아웃(Code - Data - Heap - Stack) 원칙을 유념하며 전체적인 객체 생명 주기를 관리했습니다.

## 🎮 Security Simulation Modules

| 모듈 명칭 | 대응하는 보안 위협 | 구현 핵심 기술 |
| :--- | :--- | :--- |
| **Typing** | 로그 파일 손상 및 복구 | 문자열 무결성 검증 |
| **Wire Connect** | 네트워크 물리 계층 복구 | 좌표 기반 연결 판단 로직 |
| **Code Lock** | 무차별 대입 공격(Brute-force) 대응 | 입력 횟수 제한 및 예외 처리 |
| **Slider Puzzle** | 데이터 파편화 및 복구 | 유효 이동 알고리즘 |
| **Popup Block** | 악성 팝업 및 애드웨어 차단 | 실시간 객체 생성 및 소멸 제어 |
